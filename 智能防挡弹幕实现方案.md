# 智能防挡弹幕实现方案

## Context

MyBLBL 项目使用 AkDanmaku (快手开源) 弹幕引擎，当前没有智能防挡功能。B 站官方通过 `x/player/v2` API 返回 `dm_mask.mask_url`，提供预计算的 webmask 蒙版文件（服务端人脸检测结果），客户端下载后解析 SVG 蒙版，动态调整弹幕显示区域避开人脸。零客户端计算压力。

## 数据流

```
x/player/v2 API → PlayerInfoDataWrapper.dmMask
    → VideoPlayerViewModel.loadDmMask()
        → DmMaskRepository.downloadAndParse()  [IO线程]
            → WebmaskParser 解析二进制 → List<MaskSegment>
        → DmMaskController 接收播放位置回调
            → 二分查找当前段的人脸区域
            → 计算安全显示带
            → 调整 DanmakuView 布局 + SpecialDanmakuOverlayView clipRect
```

## 实现步骤

### 阶段 1：数据层（3 个新文件 + 1 个修改）

**新建 `app/src/main/java/com/tutu/myblbl/model/dm/DmMaskInfo.kt`**
- `dm_mask` API 响应数据类（cid, plat, fps, time, maskUrl）

**新建 `app/src/main/java/com/tutu/myblbl/model/dm/WebmaskParser.kt`**
- 解析 webmask 二进制格式（参考 `.trae/bilibili-API-collect-master/docs/danmaku/webmask.md`）
- 输出 `List<MaskSegment>`（每段：timeMs + faceTopRatio + faceBottomRatio）
- 解析步骤：验证 MASK 头 → 读段表 → 逐段 gzip 解压 → 提取 SVG path 的 y 坐标范围 → 归一化为 0.0~1.0
- 纯函数，无状态

**新建 `app/src/main/java/com/tutu/myblbl/model/dm/DmMaskRepository.kt`**
- 下载 mask_url 二进制文件（复用现有网络请求能力）
- 调用 WebmaskParser 解析
- LRU 内存缓存（cid → WebmaskData，最多 4 个）
- `querySafeZone(positionMs)` 二分查找当前段的人脸区域

**修改 `app/src/main/java/com/tutu/myblbl/network/response/PlayerInfoDataWrapper.kt`**
- 添加 `@SerializedName("dm_mask") val dmMask: DmMaskInfo? = null`

### 阶段 2：ViewModel 集成

**修改 `app/src/main/java/com/tutu/myblbl/feature/player/VideoPlayerViewModel.kt`**
- 添加 `dmMaskState: StateFlow<DmMaskState>`（Idle/Loading/Ready/Unavailable）
- 在现有 `requestPlayerInfoData()` 调用后，检查 `wrapper.dmMask`
- 有 maskUrl 时启动 `loadDmMask()` 协程下载解析
- 切换视频时清除旧数据

### 阶段 3：蒙版控制器（1 个新文件）

**新建 `app/src/main/java/com/tutu/myblbl/feature/player/view/DmMaskController.kt`**
- 接收播放位置回调，查询安全显示带
- 计算策略：比较人脸上方/下方空间，选较大的作为弹幕显示带
- 防抖：安全带变化 < 5% 不更新，避免弹幕频繁跳动
- 通过修改 DanmakuView 的 LayoutParams（topMargin + height）+ `notifyDisplayerSizeChanged()` 控制普通弹幕
- 通过 SpecialDanmakuOverlayView 的 `clipRect` 控制特殊弹幕

### 阶段 4：渲染层适配

**修改 `MyPlayerDanmakuController.kt`**
- 新增 `updateMaskBand(topPx: Int, heightPx: Int)` 方法
- 实现：调整 DanmakuView 的 LayoutParams，调用 `player.notifyDisplayerSizeChanged()`

**修改 `SpecialDanmakuOverlayView.kt`**
- 新增 `updateMaskBand(topRatio: Float, bottomRatio: Float)` 方法
- 在 onDraw 中使用 `canvas.clipRect` 裁剪到安全显示带

**修改 `MyPlayerSpecialDanmakuController.kt`**
- 新增 `updateMaskBand()` 转发方法

### 阶段 5：设置开关

**修改 `MyPlayerSettingMenuBuilder.kt`**
- PanelState 添加 `dmSmartShield: Boolean`
- 弹幕设置菜单添加"智能防挡"开关项

**修改 `MyPlayerView.kt`**
- 创建 DmMaskController 实例
- SettingsSnapshot 添加 `smartShieldEnabled` 字段
- 播放位置同步时调用 `maskController.onPositionChanged()`
- 监听 dmMaskState 变化，Ready 时激活蒙版
- View 尺寸变化时通知 maskController

**修改弹幕设置持久化相关文件**
- MyPlayerSettingPreferenceStore 持久化开关状态
- SettingsFragment 全局设置页添加开关

**修改 `strings.xml`**
- 添加 `<string name="dm_smart_shield">智能防挡</string>`

## 风险与降级

1. **不是所有视频都有 dm_mask**：无 mask 时 fallback 到用户原始 screenArea 设置，体验不变
2. **AkDanmaku 不直接支持 margin**：通过调整 DanmakuView 的 LayoutParams（topMargin + height）实现，无需改 AkDanmaku 源码
3. **webmask 格式兼容性**：解析时防御性编程，header 验证失败直接返回 null
4. **性能**：webmask 下载解析在 IO 线程；查询用二分查找 O(logN)；安全带变化才更新布局（防抖）
5. **SVG 解析**：不需要完整解析 SVG，只需正则提取 `<path>` 中的 y 坐标 min/max

## 验证方式

1. 选择一个有人脸的 B 站视频（拜年祭、舞蹈区等大概率有 dm_mask）
2. 在 `DmMaskRepository.loadMask()` 打印日志确认 mask_url 非空、下载成功
3. 在 `WebmaskParser` 打印解析出的段数和人脸区域比例
4. 开启"智能防挡"开关，播放视频，观察弹幕是否避开人脸区域
5. 关闭开关，弹幕恢复原始显示区域
6. 切换视频，确认旧蒙版数据被清除
7. 选择无 dm_mask 的视频，确认不崩溃且弹幕显示正常
